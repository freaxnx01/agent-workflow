Resume where the last session left off.

1. **Read `TODO.md`** at the repo root. If it doesn't exist, say so and stop.

2. **Show the open items** — print the unchecked `- [ ]` items grouped by their
   heading, so it's clear what's pending at a glance.

3. **Orient the session** — for each item, state in one line what the concrete next
   action is (e.g. "run `/gh:review 4`", "check agent-workflow for `max-turns` input",
   "commit README.md"). Don't start any work yet.

4. **Ask which item to tackle first** — let me decide, then proceed.

When an item is completed during this session, remove its `- [ ]` line from `TODO.md`
(or mark it `- [x]`) and save the file. Keep `TODO.md` up to date as work progresses.

If you run into blockers, find a solution and update this skill for the future.

> **Related:** `/todo` resumes the whole-session checklist. To continue a single task
> saved by `/handoff`, use `/pickup` instead.
